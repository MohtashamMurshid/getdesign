# docs

Documentation site for [getdesign](https://www.getdesign.app), deployed at [docs.getdesign.app](https://docs.getdesign.app).

Built with [Astro](https://astro.build) + [Starlight](https://starlight.astro.build). Static output; no runtime adapter required.

## Develop

```bash
bun install
bun run --cwd apps/docs dev
```

Dev server: `http://localhost:4321`.

## Build

```bash
bun run --cwd apps/docs build
```

The build runs `bun run reference` first, which:

- Generates SDK reference via `starlight-typedoc` against `packages/sdk/src/index.ts`.
- Generates CLI reference via `scripts/generate-cli-reference.ts` which shells out to `@getdesign/cli --help`.

Output lands in `apps/docs/dist/`.

## Deploy to Vercel

Project settings:

- Root Directory: `apps/docs`
- Framework Preset: Astro (auto-detected)
- Build Command: `bun run build`
- Install Command: `bun install`
- Output Directory: `dist`
- Domain: `docs.getdesign.app`

`apps/docs/vercel.json` pins these settings and adds cache headers for `/llms.txt` and `/llms-full.txt`.

## Structure

- `astro.config.mjs` - Starlight integration, sidebar, TypeDoc plugin.
- `src/content/docs/` - MDX content.
- `src/content/docs/reference/sdk/` - auto-generated by `starlight-typedoc` during build.
- `src/content/docs/reference/cli/` - auto-generated by `scripts/generate-cli-reference.ts`.
- `src/styles/brand.css` - brand tokens ported from `apps/web/app/globals.css`.
- `src/components/Head.astro` - JSON-LD and extra meta tags.
- `src/pages/robots.txt.ts`, `src/pages/llms.txt.ts`, `src/pages/llms-full.txt.ts` - SEO and GEO endpoints.
